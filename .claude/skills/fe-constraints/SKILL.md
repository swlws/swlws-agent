# 前端工程约束流程 (FE Constraints Process)

本 Skill 用于引导前端开发过程中的质量控制。

## 执行流程

1. **环境分析**:
   - 参考 `references/knowledge.md` 确认文件存放位置和架构模式。
2. **编码实现**:
   - 遵循 `CODING_RULES.md` 进行代码编写。
   - 重点关注状态下移和性能优化。
3. **质量自检**:
   - 完成代码后，对照 `QUALITY_CHECK.md` 进行逐项检查。

## 适用场景

- 创建新的 React 组件或功能页面。
- 修复性能问题、闪烁问题或进行大规模重构。
- 审查现有的前端代码。
